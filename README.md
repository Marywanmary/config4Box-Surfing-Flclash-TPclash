######======================================================
######🛑 部署与排雷操作指南 (Box / Surfing / FlClash / ShellCrash 用户必读)
######======================================================
######⚠️ 致命冲突警告：
######本配置文件兼容纯血 Mihomo 内核的所有客户端，可以与 RDP / Tailscale / Rustdesk / VPN 等共存。
######但在同一台设备上，不要同时运行两套流量接管机制例如Flclash。
####
######👉 Box / Surfing 手机端操作步骤：
######1. 如果你决定使用 Box for Root / Surfing 模块 (无感静默运行)：
######   - 确保面板设置中关闭了【自动复写】。
######   - 找到地区自动代理组模板，启用⚠️ Smart 智能策略代码，注释或者删除掉⚠️ URL-Test 部分代码。
######   - 将本文件直接替换掉 Box 目录下的 config.yaml，然后重启模块。不要在手机上再打开 FlClash 去接管 VPN！
####
######👉 Flclash 手机端操作步骤：
######2. 如果你决定使用图形界面的 FlClash 客户端：
######   - 必须前往系统设置或面具/KernelSU，【关闭或卸载】Box for Root、Surfing 等底层透明代理模块。
######   - 重启手机，确保底层路由表清空。
######   - 找到地区自动代理组模板，启用⚠️ URL-Test 部分代码，注释或者删除掉⚠️ Smart 智能策略代码。
######   - 打开 FlClash，导入本文件，点击启动即可 (FlClash 会自动拉起 VPN 模式接管)。
####
######👉 Flclash 电脑端操作步骤：
######   - 找到地区自动代理组模板，启用⚠️ URL-Test 部分代码，注释或者删除掉⚠️ Smart 智能策略代码。
######   - 将本文件导入电脑端 FlClash ，开启 "TUN 模式" 或 "系统代理" 即可裸跑。
####
######👉 ShellCrash 路由端操作步骤：
######   - 找到tproxy-port: 7895，将其改为tproxy-port: 9898，
######   - 导入配置，按提示操作。（目前作者尚未实测）

######🔰 新手必读：代理大区选择指南 (如何选择最优节点)
######------------------------------------------------------
######🇭🇰 香港 (HK)：物理距离最近，延迟极低。适合日常网页浏览、游戏加速。⚠️ 缺点：风控极严，ChatGPT、TikTok 等通常会封锁香港 IP。
######🇸🇬 狮城 (SG)：亚太核心枢纽。原生 IP 多，解锁流媒体能力极强（Netflix/Disney+），连接 Telegram 极速，风控相对宽松，推荐作为主力大区。
######🇹🇼 台湾 (TW)：万能平衡点。速度快，能完美解锁各类区域限制内容，且 ChatGPT 等 AI 工具基本不封锁。
######🇯🇵 日本 (JP)：亚洲互联网枢纽。适合二次元文化站(DMM/Pixiv)解锁，UDP 连通性好，适合语音通话(Discord)。
######🇺🇸 美国 (US)：终极解锁方案。延迟较高(150ms+)，但遇到极其严格的区域限制(如 Claude、美国本土银行、原生 TikTok)时，请手动切至美国。
######🇰🇷 韩国 (KR)：北方宽带(如联通)的物理外挂，延迟极低；拥有独占韩服游戏与本土流媒体解锁能力。⚠️ 缺点：路由两极分化严重，非原生 IP 风控极严，建议作为“战术备用”，不推荐作全局主力。
######🇷🇺 俄罗斯 (RU)：特殊冷门大区。适合特定区域资源与网站访问，日常极少使用。
