# 👉 Box / Surfing 手机端操作步骤：
# 1. 如果你决定使用 Box for Root / Surfing 模块 (无感静默运行)：
#    - 确保面板设置中关闭了【自动复写】。
#    - 找到地区自动代理组模板，启用⚠️ Smart 智能策略代码，注释或者删除掉⚠️ URL-Test 部分代码。
#    - 将本文件直接替换掉 Box 目录下的 config.yaml，然后重启模块。不要在手机上再打开 FlClash 去接管 VPN！
#
# 👉 Flclash 手机端操作步骤：
# 2. 如果你决定使用图形界面的 FlClash 客户端：
#    - 必须前往系统设置或面具/KernelSU，【关闭或卸载】Box for Root、Surfing 等底层透明代理模块。
#    - 重启手机，确保底层路由表清空。
#    - 找到地区自动代理组模板，启用⚠️ URL-Test 部分代码，注释或者删除掉⚠️ Smart 智能策略代码。
#    - 打开 FlClash，导入本文件，点击启动即可 (FlClash 会自动拉起 VPN 模式接管)。
#
# 👉 Flclash 电脑端操作步骤：
#    - 找到地区自动代理组模板，启用⚠️ URL-Test 部分代码，注释或者删除掉⚠️ Smart 智能策略代码。
#    - 将本文件导入电脑端 FlClash ，开启 "TUN 模式" 或 "系统代理" 即可裸跑。
#
# 👉 ShellCrash 路由端操作步骤：
#    - 找到tproxy-port: 7895，将其改为tproxy-port: 9898，

# 🔌 面向新手的使用指南（面板开关 vs 配置文件）
# ======================================================
# ❓ 核心疑问：如果我在手机 App（如 FlClash）或电脑面板上切换了 TUN / TProxy / Redirect / Mixed / Enhance 等模式，我需要回来修改下面这些代码吗？
# 💡 终极答案：【绝对不需要修改任何一行代码！】
#
# 📖 原理说明：
# 这份配置文件的“底子”已经打得完美且全能。这里的代码相当于预先造好了所有模式的“大门”（开放了相应的端口和规则）。
# 当你在 GUI 界面上拨动模式开关时，App 会直接通过底层 API 在【内存】中发号施令，把流量精准导向对应的门。它无需篡改这里的 YAML 文本。
#
# 1. 关于 TUN 模式（虚拟网卡）：
# 看到下方写着 `tun: enable: false` 不要惊慌。这是为了保护底层 Root 玩家防止“双重劫持冲突”。作为普通用户，只要你在 App 界面上点击开启 VPN，App 就会在运行内存里强行拉起虚拟网卡，稳稳接住你的所有流量。下方代码一个字都不用改。
#
# 2. 关于 Mixed 模式（局域网共享/主机加速）：
# 预设已完美开启 `allow-lan: true` 和端口 `7890`。如果你需要给同一 WiFi 下的 Switch 或 PC 共享网络，直接在对方设备填入本机内网 IP 和 7890 端口即可。
#
# 3. 关于 ENHANCE（DNS Fake-IP 模式）：
# 预设已在底层开启。全权由你的面板“代理模式”调度：面板切到“规则/全局”时，它自动发威秒开网页；面板切“直连”时，它自动休眠。
#
# 4. 关于 TProxy / Redirect 模式（高级 Root 劫持）：
# 预设接管端口已备好（9898 / 7892）。无论 Box for Root / Surfing 脚本推哪种底层的防火墙流量进来，内核都能开门放行。
